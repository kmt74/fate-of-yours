# 🔮 Fate of Yours

> *An AI-powered tarot reading experience — mystical, multilingual, and deeply personal.*

**Fate of Yours** is a premium web application that combines classical tarot wisdom with the power of Google Gemini AI. Users draw three cards representing **Past, Present, and Future**, receive richly formatted AI interpretations, and revisit their readings through a curated history journal.

---

## ✨ Features

- **Three-Card Spread** — Draw cards for Past, Present & Future in an immersive animated interface
- **AI-Powered Readings** — Deep, personalized interpretations generated by Google Gemini
- **Tarot Deck Selector** — Choose your preferred deck before each reading
- **Reading History** — Save, browse, and revisit all past readings
- **Multilingual Support** — Full UI in 🇬🇧 English, 🇻🇳 Vietnamese, and 🇨🇳 Chinese
- **Admin Dashboard** — Secure analytics portal with real-time usage stats and session tracking
- **Responsive & Animated** — Glassmorphism design with GSAP & Framer Motion animations

## 🚀 Getting Started

### Prerequisites

- **Node.js** ≥ 18
- **npm** ≥ 9
- A [Supabase](https://supabase.com) project
- A [Google Gemini API](https://aistudio.google.com/app/apikey) key

### 1. Clone the repository

```bash
git clone https://github.com/your-username/fate-of-yours.git
cd fate-of-yours
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Then edit `.env` and fill in your credentials:

```env
# Supabase
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_public_key
SUPABASE_URL=your_supabase_project_url
SUPABASE_SERVICE_KEY=your_supabase_service_role_key

# Google Gemini
GEMINI_API_KEY=your_gemini_api_key
VITE_GEMINI_API_KEY=your_gemini_api_key
```

### 4. Run locally

#### Frontend only (Vite dev server)
```bash
npm run dev
```
> App available at `http://localhost:5173`

#### Frontend + Express proxy (recommended)
```bash
npm run dev:all
```
> Vite on `:5173`, Express proxy on `:5000`

#### Via Vercel CLI (closest to production)
```bash
npm run local
```

## 🔑 Environment Variables Reference

| Variable | Used By | Description |
|---|---|---|
| `VITE_SUPABASE_URL` | Frontend | Supabase project URL |
| `VITE_SUPABASE_ANON_KEY` | Frontend | Supabase public anon key |
| `SUPABASE_URL` | Scripts | Supabase project URL (server-side) |
| `SUPABASE_SERVICE_KEY` | Scripts | Supabase service role key (secret!) |
| `GEMINI_API_KEY` | API / Server | Gemini key for serverless functions |
| `VITE_GEMINI_API_KEY` | Frontend | Gemini key for client-side calls |

> ⚠️ **Never commit your `.env` file.** It is already listed in `.gitignore`.

---

## 🔒 Admin Access

The admin portal (`/admin`) is protected by a session-based authentication flow:

1. Navigate to `/admin/login`
2. Enter admin credentials
3. On success, `isAdminAuthenticated` is written to `sessionStorage`
4. Accessing `/admin` without authentication redirects to `/admin/login`
